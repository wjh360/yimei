## 开发规范

### 1. 页面结构
- 使用 `<template>`, `<script>`, `<style>` 三段式结构
- style 标签必须加 scoped 属性和 lang="scss"
- 引入全局样式文件

### 2. 命名规范
- 文件名：kebab-case（短横线）
- 组件名：PascalCase（大驼峰）
- 变量名：camelCase（小驼峰）
- 样式类名：kebab-case（短横线）

### 3. 样式规范
- 使用全局样式变量
- 遵循样式指南中的设计规范
- 使用 SCSS 预处理器
- 保持视觉一致性

### 4. 表单验证
- 使用 FormValidator 工具类
- 统一的验证规则格式
- 统一的错误提示方式

### 5. 接口调用
- 遵循 API 文档规范
- 统一的错误处理
- 统一的加载状态处理

### 6. 代码风格
- 使用 ES6+ 语法
- 使用 async/await 处理异步
- 适当的代码注释
- 合理的代码分层

## 开发流程

1. 新建页面
   - 复制模板页面
   - 修改页面名称和路由
   - 调整页面结构

2. 开发功能
   - 引入必要的组件和工具
   - 实现业务逻辑
   - 添加表单验证
   - 实现接口调用

3. 样式调整
   - 应用全局样式
   - 添加页面特定样式
   - 确保响应式布局

4. 测试验证
   - 功能测试
   - 样式兼容性测试
   - 表单验证测试
   - 接口联调测试

## 注意事项

1. 性能优化
   - 合理使用计算属性
   - 避免不必要的渲染
   - 图片资源优化

2. 代码质量
   - 代码复用
   - 及时重构
   - 保持简洁

3. 用户体验
   - 合适的加载状态
   - 友好的错误提示
   - 平滑的动画效果
